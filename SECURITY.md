# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅ Yes    |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities privately:

1. Email: security reports should be sent directly to the repository maintainer via GitHub's private vulnerability reporting feature.
2. Go to **Security → Report a vulnerability** in this repository.
3. Provide: description, reproduction steps, impact assessment, and any suggested fixes.

You will receive a response within 72 hours. Please allow time for a patch before public disclosure.

## Scope

This tool is intended for authorized security testing and research only. Misuse against systems you do not own or have explicit permission to test is out of scope and strictly prohibited.
